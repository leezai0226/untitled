# 마이페이지 디지털 상품 다운로드 상태 DB 동기화 및 환불 버그 수정 요청

현재 마이페이지에서 디지털 상품(shop 구매 건)을 다운로드하면 화면상에서는 환불 버튼이 비활성화되지만, 로그아웃 후 재로그인하거나 페이지를 새로고침하면 다시 환불 버튼이 활성화되는 치명적인 버그가 있어. 
상태가 프론트엔드의 로컬 state에만 머물러 있고 DB와 동기화되지 않아서 발생하는 문제로 보이니, 아래 단계에 따라 완벽하게 수정해 줘.

## 1. 데이터베이스(Supabase) 업데이트 로직 점검 및 추가
- 사용자가 [다운로드] 버튼을 클릭하는 순간, 단순히 파일만 다운로드되는 것이 아니라 비동기 API 통신(또는 Server Action)을 통해 Supabase의 주문/구매 테이블(`Orders` 등)에 있는 해당 주문 건의 `is_downloaded` 값을 `true`로 확실하게 업데이트하는 로직을 추가해 줘.

## 2. 마이페이지 데이터 패칭(Fetching) 로직 수정
- 마이페이지(`/app/mypage/page.tsx` 등)가 렌더링될 때, 단순히 결제 내역만 가져오는 것이 아니라 DB에 저장된 `is_downloaded` (다운로드 여부) 컬럼의 최신 값을 정확하게 함께 불러오도록 쿼리문을 수정해 줘.

## 3. 프론트엔드 UI 렌더링 조건 수정
- [환불하기] 버튼의 활성화/비활성화 상태를 프론트엔드의 임시 state가 아니라, 방금 DB에서 불러온 `is_downloaded` 값에 절대적으로 의존하도록 수정해 줘.
- `is_downloaded === true`라면, 재로그인을 하든 새로고침을 하든 무조건 [환불하기] 버튼이 비활성화(disabled) 상태로 렌더링되게 만들어 줘.
