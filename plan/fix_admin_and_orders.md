# 관리자 페이지 고도화 및 구매/주문 내역 버그 수정 요청

현재 웹사이트의 마이페이지 구매 내역 표시 오류를 해결하고, 관리자 페이지의 보안 및 주문 관리 UI를 대폭 개선하려고 해. 아래 3가지 카테고리의 문제들을 완벽하게 수정해 줘.

## 1. 마이페이지 구매 내역 표시 오류 수정 (버그 픽스)
- **증상:** 코드를 수정하는 과정에서 마이페이지(구매 내역)에 사용자가 구매한 품목(디지털 에셋 등)이 계속 보이지 않는 렌더링 오류가 발생했어.
- **해결 방안:** 프론트엔드의 데이터 패칭 로직과 Supabase 쿼리문을 점검해서, 사용자의 UID와 일치하는 주문(Orders) 데이터가 정상적으로 불러와지고 화면에 렌더링되도록 코드를 수정해 줘.

## 2. 관리자 페이지 접근 권한(Admin Guard) 철저한 분리
- **요구사항:** 관리자 페이지(`/app/admin/...`)는 오직 지정된 내 최고 관리자 계정(이메일 + 비밀번호)으로 로그인했을 때만 접근할 수 있어야 해.
- **해결 방안:** - 미로그인 사용자나, 로그인했더라도 관리자 이메일이 아닌 일반 사용자 계정으로 `/admin` 경로에 접근하려고 하면 즉시 접근을 차단하고 메인 홈(`/`)이나 로그인 페이지(`/login`)로 강제 리다이렉트 시키는 미들웨어(Middleware) 또는 Layout 렌더링 가드 로직을 구현해 줘.

## 3. 관리자 페이지 주문 관리 화면(UI/UX) 대폭 개선
현재 주문 관리 리스트에서 발생하는 불편한 점들을 아래와 같이 개선해 줘.

- **카테고리 탭 분리:** 모든 주문이 섞여서 보이지 않도록, 상단에 [클래스 주문] / [Shop 주문(디지털 에셋)] 탭을 만들어서 각각 분류해서 볼 수 있게 해 줘.
- **UI 줄바꿈 방지:** 테이블 내의 '이름'이나 '결제완료' 같은 짧은 텍스트 항목이 좁은 칸 때문에 2줄로 찌그러져 나오는 현상이 있어. `whitespace-nowrap` 등의 Tailwind CSS를 적용해서 무조건 한 줄로 깔끔하게 떨어지게 뷰포트를 조정해 줘.
- **상품 타입 분류 버그 픽스:** Shop에서 디지털 에셋을 구매했는데도 관리자 페이지에서는 '클래스 주문'이라고 잘못 표시되는 논리적 오류가 있어. 주문된 상품의 카테고리나 DB의 `type` 값을 정확히 판별하여 올바른 텍스트가 출력되게 고쳐 줘.
- **결제 수단 표시:** 포트원 결제 데이터(`pay_method` 등)를 바탕으로, 카드로 결제한 경우 목록에 '카드 결제' (또는 간편결제 등)라는 수단이 명시되도록 컬럼을 하나 추가해 줘.
- **주문 내역 삭제(숨김) 기능:** 테스트 결제 건이나 관리자가 더 이상 볼 필요 없는 주문 내역의 맨 우측에 **[X] 버튼(삭제 아이콘)**을 만들어 줘. 클릭 시 DB에서 해당 주문을 삭제하거나(Hard Delete) 숨김 처리(Soft Delete, `is_hidden: true` 추가)하여 관리자 화면에서 즉시 사라지게 만들어 줘.
