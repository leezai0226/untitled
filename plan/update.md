# 톤앤매너를 유지한 동적 멀티 링크 메인 페이지 및 관리자 기능 구현

메인 페이지(`/app/page.tsx`)를 모바일 친화적인 멀티 링크(Linktree) 구조로 개편하되, 기존 사이트의 디자인 톤앤매너를 그대로 유지하고 모든 링크를 관리자 페이지에서 제어할 수 있도록 아래 사항을 구현해 줘.

## 1. 데이터베이스(Supabase) 테이블 신규 생성
- 메인 페이지의 링크 정보를 관리할 `main_links` 테이블을 생성해 줘.
- **필요한 컬럼:** `id` (UUID), `title` (버튼 이름), `url` (이동할 주소), `is_active` (노출 여부 boolean), `sort_order` (정렬 순서 int), `created_at`.

## 2. 관리자 페이지 '메인 링크 관리' 기능 추가 (`/app/admin/links/...`)
- 관리자 대시보드 사이드바에 **[메인 링크 관리]** 메뉴를 추가해 줘.
- 해당 페이지에서 새로운 링크를 [추가], 기존 링크의 이름/주소를 [수정], 불필요한 링크를 [삭제] 또는 [숨김(is_active 토글)] 처리할 수 있는 표(Table) 형태의 UI를 만들어 줘.
- 위아래 화살표 버튼이나 드래그 앤 드롭을 통해 메인 화면에 노출될 버튼의 순서(`sort_order`)를 변경할 수 있도록 해 줘.

## 3. 메인 페이지 프론트엔드 개편 (`/app/page.tsx`)
- 메인 페이지 접속 시, `main_links` 테이블에서 `is_active`가 true인 데이터만 `sort_order` 순서대로 불러와서 렌더링해 줘 (Server Component 활용 권장).
- **디자인 요구사항 (가장 중요!):** - 저렴해 보이는 템플릿 느낌이 나지 않도록, 현재 우리 웹사이트(`tailwind.config.ts` 등)에 정의된 **기존의 프라이머리 컬러, 폰트 굵기, 둥근 모서리(border-radius) 스타일을 그대로 상속**받아 적용해 줘.
  - 최대 너비는 모바일 환경에 최적화(`max-w-md mx-auto`)하되, 배경과 버튼의 색상 대비가 사이트의 기존 분위기와 완벽히 어우러지게 CSS를 다듬어 줘.
  - 최상단 프로필 이미지(또는 로고) 영역과 "동그란문" 타이틀, 한 줄 소개 텍스트 영역도 우리 브랜드에 맞게 깔끔하게 배치해 줘.

## 4. 초기 더미 데이터 세팅 (선택)
- 코드가 적용된 후 화면이 비어 보이지 않도록, DB 마이그레이션 시 아래 2개의 기본 링크 데이터를 세팅해 줘.
  1. 클래스 수강신청 (`/class`)
  2. 디지털 에셋 Shop (`/shop`)
