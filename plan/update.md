# 쇼핑몰/클래스 결제 시스템 버그 수정 및 UI 개선 요청

테스트 과정에서 발견된 환불 로직, 계좌이체 처리, 그리고 UI 배치와 관련된 4가지 문제점을 아래 지시사항에 따라 완벽하게 수정해 줘.

## 1. Shop 상품 환불 상태 DB 동기화 (새로고침 버그 수정)
- **증상:** 마이페이지에서 Shop 상품을 환불 처리했는데, 새로고침(F5)을 하면 환불된 상품이 다시 정상 상태(또는 리스트에 다시 나타나는 현상)로 렌더링되고 있어.
- **해결 방안:** 환불 로직 실행 시 Supabase `Orders` 테이블의 상태(status)가 `refunded`나 `cancelled`로 확실하게 업데이트되도록 백엔드 코드를 점검해 줘. 또한, 마이페이지에서 데이터를 불러올 때 임시 state가 아닌 DB의 최신 상태값을 기준으로 환불 완료된 화면(버튼 비활성화 또는 취소선 등)을 정확히 렌더링해 줘.

## 2. 계좌이체 결제 건 마이페이지 노출 조건 수정
- **증상:** 카드가 아닌 '계좌이체(무통장입금)'로 결제한 경우, 관리자가 '입금 확인'을 누르기 전까지는 사용자의 마이페이지 구매 내역에 아예 안 보이는 현상이 있어. 
- **해결 방안:** 사용자는 돈을 보내고 확인을 기다리는 중인데 내역이 없으면 당황할 수밖에 없어. 마이페이지 데이터를 패칭할 때 `status === 'completed'`인 주문뿐만 아니라, `status === 'pending'`(입금 대기/결제 대기) 상태인 주문도 같이 불러와 줘. 단, UI 상에 **[입금 확인 중]** 이라는 뱃지나 텍스트를 띄워서 상태를 명확히 안내해 줘.

## 3. 관리자 페이지 계좌이체 수동 환불 승인 기능 추가
- **증상:** 계좌이체 결제 건은 포트원 API로 자동 결제 취소가 불가능한 경우가 많아 관리자가 직접 송금 후 수동으로 처리해야 하는데, 관리자 페이지에 이를 승인할 버튼이 없어.
- **해결 방안:** 관리자 페이지(`/app/admin/orders` 등) 주문 관리 리스트에서, 사용자가 '환불 신청'을 한 '계좌이체' 주문 건에 한해 **[환불 승인]** 버튼을 노출해 줘. 이 버튼을 클릭하면 수동으로 DB의 주문 상태를 `refunded`로 변경 처리하는 로직을 추가해 줘.

## 4. Shop 상세 페이지 미디어 슬라이더 위치 변경
- **증상:** 최근 추가한 사진+동영상 슬라이더 컴포넌트가 상품 상세 페이지(`/app/shop/[id]/page.tsx`)의 맨 위쪽(최상단)에 배치되어 있어 레이아웃이 어색해.
- **해결 방안:** 슬라이더 컴포넌트의 렌더링 위치를 최상단에서 **'상세 이미지(본문 내용)가 시작되는 영역 바로 위'**로 이동시켜 줘. (보통 상품명/가격/장바구니 버튼 등이 있는 상품 요약 섹션의 바로 아래가 적당함)
