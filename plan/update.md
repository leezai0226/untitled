# 비회원 결제 허용 및 이메일 자동 배송 시스템 구현 요청

로그인 없이도 상품(Shop/Class)을 구매할 수 있는 '비회원 결제' 기능을 도입하고, 결제 완료 시 입력한 이메일로 콘텐츠를 자동 발송하는 로직을 구현해 줘.

## 1. 프론트엔드: 로그인 가드 해제 및 정보 입력 폼 추가
- **접근 권한 수정:** 상세 페이지에서 [구매하기] 또는 [수강신청] 버튼 클릭 시 로그인 페이지로 리다이렉트 시키던 로직을 제거해 줘.
- **비회원 정보 입력:** 비회원일 경우, 결제창이 뜨기 전에 '결제 결과 및 상품을 받을 이메일 주소'와 '연락처'를 입력받는 간단한 모달이나 폼을 띄워 줘.
- **포트원 연동:** 입력받은 이메일을 포트원 `IMP.request_pay`의 `buyer_email` 파라미터로 정확히 전달해 줘.

## 2. 데이터베이스(Supabase) 스키마 수정
- `Orders` 테이블에서 `user_id` 컬럼을 **Nullable(선택 사항)**로 변경해 줘. (비회원은 UID가 없으므로)
- 대신, 비회원 식별을 위해 `guest_email`과 `guest_phone` 컬럼을 주문 정보에 함께 저장하도록 수정해 줘.

## 3. 백엔드: 결제 검증 및 이메일 자동 발송 로직 (`/api/portone/verify/route.ts`)
- **결제 검증:** 포트원 결제 검증이 성공(`status === 'completed'`)하면, 해당 주문이 비회원인지 확인해 줘.
- **이메일 발송 트리거:** 검증 성공 즉시, 결제된 상품이 '디지털 에셋(Shop)'이면 다운로드 링크를, '클래스'라면 수강 안내 및 단톡방 링크 등을 결제 시 입력한 이메일로 자동 발송해 줘.
- **Nodemailer/Resend 연동:** 기존에 설정한 이메일 발송 기능을 활용하여, 비회원에게는 별도의 '구매 확인 및 상품 전달' 메일이 가도록 템플릿을 구성해 줘.

## 4. 마이페이지 접근 및 예외 처리
- 비회원은 로그인이 없으므로 마이페이지 접근이 불가해. 따라서 메일 본문에 "주문 번호"와 함께 "추후 회원가입 시 동일한 이메일로 가입하면 내역이 통합됩니다"라는 안내를 넣어주거나, 이메일로 받은 링크를 통해서만 다운로드가 가능하도록 처리해 줘.

# 카카오 로그인 기능 및 UI 완전 제거 요청

비회원 결제 및 로그인 시스템 간소화를 위해, 현재 프로젝트에 적용되어 있는 '카카오 로그인' 기능을 완전히 제거해 줘.

## 1. 프론트엔드 UI 제거 (`/app/login/page.tsx` 또는 Auth 컴포넌트)
- 로그인/회원가입 화면에서 '카카오로 로그인' 버튼 UI를 완전히 삭제해 줘.
- 카카오 버튼이 빠진 후 빈 공간이 어색하지 않도록, 남은 '구글로 로그인' 버튼의 너비나 여백(margin/padding)을 조정해서 화면 중앙에 깔끔하게 배치해 줘.

## 2. 인증 로직 및 불필요한 코드 정리
- Supabase Auth를 통해 카카오 로그인을 실행하던 함수(예: `signInWithOAuth({ provider: 'kakao' })`)와 관련된 코드를 모두 지워 줘.
- 카카오 로그인과 관련된 환경 변수나 불필요해진 import 구문이 남아있다면 찌꺼기가 없도록 파일들을 점검하고 정리해 줘.
