# 마이페이지 고도화 및 결제/환불 핵심 비즈니스 로직 적용 요청

현재 웹사이트의 마이페이지 기능과 환불, 알림 시스템을 자동화하기 위해 아래 4가지 사항을 구현해 줘.

## 1. 마이페이지 내 수강신청 내역 조회 기능
- 사용자가 결제한 클래스(브이로그반, 숏폼반 등) 수강신청 내역을 마이페이지(`/app/mypage/page.tsx`)에서 확인할 수 있도록 Supabase와 연동하여 리스트 형태로 보여주는 UI를 추가해 줘.

## 2. 클래스 환불 정책에 따른 로직 및 UI 제어
- 클래스 수강 내역에 [환불하기] 버튼을 추가하고, '현재 시간'과 '수강 진행일(시작일)'의 날짜 차이를 계산하여 아래 정책을 시스템적으로 강제해 줘. 시간은 전부 대한민국 서울 기준으로 설정해서.
  - **7일 전까지:** 100% 환불 가능
  - **5일 전까지 (5~6일 전):** 결제 금액의 50% 부분 환불 가능
  - **4일 이내 (4일~당일):** 환불 불가 ([환불하기] 버튼 비활성화 및 안내 문구 노출)
  - **예외 없는 적용:** 오늘 구매했더라도 수강일이 당장 내일이라면 4일 이내 조건에 해당하므로 즉시 환불이 불가하도록 날짜 계산 로직을 꼼꼼하게 작성해 줘.

## 3. 디지털 에셋 환불 버그 수정, 권한 설정 및 실제 포트원 결제 취소 연동
- **DB 상태 유지 버그 수정:** 현재 shop에서 상품 다운로드 시 UI에서는 환불이 막히지만, 로그아웃 후 재로그인 시 환불 버튼이 다시 활성화되는 현상이 있어. Supabase의 `is_downloaded` 상태가 정확히 업데이트되고 렌더링 시 제대로 반영되도록 고쳐 줘.
- **마이페이지 접근 권한:** 로그아웃 상태에서 `/app/mypage`로 접근할 경우, 접속을 차단하고 `/login` 페이지로 리다이렉트(Middleware 또는 Auth Guard 활용) 시켜 줘. 마이페이지에서 로그아웃을 할 경우에도 바로 `/login` 페이지로 이동할 수 있도록 해줘
- **실제 결제 취소 (포트원 연동):** [환불하기] 버튼은 단순 UI 변경이 아니라, 포트원(PortOne) REST API의 '결제 취소 API'를 호출하는 서버리스 함수(`/api/portone/cancel` 등)와 연결되어야 해. 100% 취소와 클래스의 50% 부분 취소 로직을 모두 구현해 줘.

## 4. 결제 완료 시 관리자 이메일 알림 발송 기능
- 고객의 결제가 정상적으로 완료되고 DB 업데이트가 끝나면, 관리자인 내 이메일로 "새로운 결제가 완료되었습니다(상품명, 금액 등)"라는 알림 메일이 자동으로 발송되도록 백엔드 로직을 추가해 줘.
- Next.js 환경에서 사용하기 적합한 Nodemailer나 Resend(추천) 패키지를 활용하여 세팅해 줘.
